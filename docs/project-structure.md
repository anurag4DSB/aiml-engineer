# Project Structure

This guide explains the organization of the aiml.engineer repository.

## Directory Layout

```text
aiml-engineer/
├── content/                 # Site content (posts, notes, labs, etc.)
│   ├── index.md            # Homepage
│   ├── posts/              # Blog posts
│   ├── notes/              # Technical notes
│   ├── labs/               # Experiments and notebooks
│   ├── projects/           # Project case studies
│   └── reading/            # Paper summaries
├── docs/                    # Repository documentation (this directory)
│   ├── README.md           # Documentation index
│   ├── setup.md            # Setup guide
│   ├── linting.md          # Linting guide
│   ├── content-guide.md    # Content guide
│   ├── building.md         # Building guide
│   ├── ci-cd.md            # CI/CD guide
│   ├── project-structure.md # This file
│   └── features.md         # Features guide
├── .github/
│   └── workflows/
│       └── publish-docs.yml # CI/CD workflow
├── site/                    # Generated static site (gitignored)
├── .venv/                   # Python virtual environment (gitignored)
├── mkdocs.yml              # MkDocs configuration
├── pyproject.toml          # Python project metadata & dependencies
├── uv.lock                 # Locked dependency versions
├── .pre-commit-config.yaml # Pre-commit hooks configuration
├── .yamllint.yml           # YAML linting rules
├── CONTRIBUTING.md         # Quick contribution guide
├── README.md               # Project README
├── NOTICE.md               # Third-party notices
└── CNAME                   # Custom domain for GitHub Pages
```

## Key Directories

### content/

**Purpose:** All site content (blog posts, notes, labs, projects, reading)

**Structure:**

- Each section has an `index.md` file as landing page
- Individual content files in the same directory
- Posts use date-prefixed names: `YYYY-MM-DD-title.md`
- Other content uses kebab-case: `descriptive-name.md`

**Renamed from:** `docs/` (changed to reserve "docs" for repository documentation)

### docs/

**Purpose:** Repository documentation for contributors and maintainers

**Structure:**

- Flat structure with all `.md` files at root level
- `README.md` serves as navigation hub
- Each file covers a single topic

**Note:** This is different from the `content/` directory which contains the published site content.

### .github/workflows/

**Purpose:** GitHub Actions CI/CD workflows

**Files:**

- `publish-docs.yml` - Lint, build, and deploy the site on push to main

### site/

**Purpose:** Generated static site output (not committed to git)

**Generated by:** `mkdocs build`

**Contains:** HTML, CSS, JS, images, and search index

## Key Files

### mkdocs.yml

**Purpose:** MkDocs configuration

**Contains:**

- Site metadata (name, URL, description)
- Theme configuration (Material theme, colors, features)
- Navigation structure
- Plugins (search, git-revision-date, minify)
- Markdown extensions
- Content directory (`docs_dir: content`)

### pyproject.toml

**Purpose:** Python project configuration and dependencies

**Sections:**

- `[project]` - Project metadata (name, version, license, authors)
- `[dependencies]` - Main dependencies (mkdocs, mkdocs-material, plugins)
- `[dependency-groups]` - Dev dependencies (pre-commit, yamllint, pymarkdownlnt)
- `[tool.pymarkdown]` - Markdown linting rules

### uv.lock

**Purpose:** Locked dependency versions for reproducible builds

**Generated by:** `uv sync`

**Important:** Always commit this file alongside `pyproject.toml` changes

### .pre-commit-config.yaml

**Purpose:** Pre-commit hooks configuration

**Hooks:**

- Trailing whitespace removal
- End-of-file fixer
- YAML validation
- Large file detection
- Merge conflict detection
- Markdown linting (pymarkdown)
- YAML linting (yamllint)

### .yamllint.yml

**Purpose:** YAML linting rules

**Configuration:**

- Max line length: 120 characters
- Indentation: 2 spaces
- Ignored paths: `.venv/`, `site/`, `uv.lock`

### CNAME

**Purpose:** Custom domain configuration for GitHub Pages

**Contains:** `aiml.engineer`

**Note:** Maintained by the deployment workflow

## Content Organization

### Posts (Blog)

**Location:** `content/posts/`

**Naming:** `YYYY-MM-DD-title.md`

**Purpose:** Time-based blog posts, announcements, updates

**Example:**

```text
content/posts/
├── index.md
├── 2025-09-26-hello-world.md
└── 2025-12-15-building-rag-system.md
```

### Notes

**Location:** `content/notes/`

**Naming:** `descriptive-name.md`

**Purpose:** Technical notes, tutorials, how-tos, evergreen content

**Example:**

```text
content/notes/
├── index.md
├── transformer-architecture.md
└── fine-tuning-llama3.md
```

### Labs

**Location:** `content/labs/`

**Naming:** `descriptive-name.md`

**Purpose:** Experiments, notebooks, proof-of-concepts

**Example:**

```text
content/labs/
├── index.md
├── rag-evaluation.md
└── prompt-engineering-experiments.md
```

### Projects

**Location:** `content/projects/`

**Naming:** `descriptive-name.md`

**Purpose:** Project case studies, portfolio pieces

**Example:**

```text
content/projects/
├── index.md
├── llm-powered-chatbot.md
└── document-qa-system.md
```

### Reading

**Location:** `content/reading/`

**Naming:** `descriptive-name.md`

**Purpose:** Paper summaries, literature reviews

**Example:**

```text
content/reading/
├── index.md
├── attention-is-all-you-need.md
└── rag-improvements.md
```

## Configuration Files

### Tool Configuration

**pymarkdown:** `pyproject.toml` → `[tool.pymarkdown]`

**yamllint:** `.yamllint.yml`

**pre-commit:** `.pre-commit-config.yaml`

**mkdocs:** `mkdocs.yml`

### Git Configuration

**Ignored files** (`.gitignore`):

- `.venv/` - Virtual environment
- `site/` - Generated site
- `__pycache__/` - Python cache
- `*.pyc` - Compiled Python files
